# Task Manager
## Технологии

 C#, ASP.NET Core, Entity Framework Core, PostgreSQL

 ## Запуск проекта

 Проект создан на net.Core 10.0
 Проект запускается либо из VS2026 по клавишам CTRL+F5 либо c помощью VS Code из терминала с использованием команды dotnet run
 Перед запусом необходимо преверить сборки и наличие Базы Данных Postgres

  ## Архитектурные решения
  В основу проекта положена луковая архитектура с разделением на слои.
  В данном случае это позволить протестировать каждый слой. Также применен автомеппер при созданнии соответвий между слоями. 
  Сервисы прокинуты в контроллеры через интерфейсы с использованием подхода DI. 
  В дальнейшем можно несколко упростить слой Service и использовать для стандартных GRUD операций универсальный класс Service<T>. 


  ## Слои приложения
  AppDataContext: Эта папка будет содержать контекст базы данных, который отвечает за взаимодействие с базой данных.
  Сontracts: В этой папке будут размещены наши объекты передачи данных (DTO), которые используются для формирования данных, передаваемых между клиентом и сервером.
  Models: Эта папка будет содержать модели, которые представляет структуру проекта.
  Controllers: В этой папке будет размещены контроллеры, которые обрабатывают входящие HTTP-запросы и отправляет ответы.
  Interfaces: Эта папка будет содержать интерфейсы, которые определяет контракты для классов service.
  Services: В этой папке будет размещен классы Service, который реализуют интерфейсы IService и содержат бизнес-логику нашего приложения.
  Mapping: Эта папка будет содержать профиль сопоставления, который используется для сопоставления свойств между различными объектами.
  Middleware: В этой папке будет размещено промежуточное программное обеспечение exception, которое обрабатывает исключения глобально в нашем приложении.
  Migration: В этой папке содержатся все миграции. 

  При масштабировании проекта можно будет разнести слои по отдельным проектам для улучшения читаемости. 
  Использование для такого проекта подохода CQRS - кажется излишним.
